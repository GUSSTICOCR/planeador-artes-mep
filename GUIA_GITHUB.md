# 🌐 Guía para Subir tu Proyecto a GitHub

Subir tu proyecto a GitHub es el primer paso para que tu aplicación sea accesible desde cualquier parte del mundo. Sigue estos pasos:

## 1. Crear tu cuenta en GitHub
1.  Ve a [github.com](https://github.com/).
2.  Haz clic en **"Sign up"** y crea tu cuenta con tu correo electrónico.

## 2. Crear un nuevo Repositorio
Un "Repositorio" es como una carpeta en la nube para tu proyecto.
1.  En tu página de inicio de GitHub, haz clic en el botón verde **"New"** (o en el símbolo **+** arriba a la derecha y luego **"New repository"**).
2.  **Repository name**: Ponle algo como `planeador-artes-mep`.
3.  **Public/Private**: Déjalo en **Public** si quieres que sea fácil de compartir, o **Private** si solo tú quieres verlo.
4.  No marques ninguna otra casilla (ni README, ni .gitignore, ni license, porque ya los tenemos).
5.  Haz clic en el botón verde **"Create repository"**.

## 3. Subir los archivos (Método Simple)
Como no tienes Git instalado aún, el método más rápido para empezar es este:
1.  En la pantalla que aparece tras crear el repositorio, busca el enlace que dice **"uploading an existing file"**.
2.  **¡IMPORTANTE!**: Abre la carpeta de tu proyecto en tu computadora.
3.  Selecciona **TODOS** los archivos **MENOS** la carpeta `node_modules`.
    -   Selecciona: `galeria.html`, `server.js`, `package.json`, `.gitignore`, `INSTRUCCIONES.md`, etc.
4.  Arrástralos todos a la ventana de GitHub en tu navegador.
5.  Espera a que carguen y haz clic en el botón verde **"Commit changes"** al final de la página.

---

### 🎨 ¿Qué sigue después de esto?
Una vez que tus archivos estén en GitHub, avísame. El siguiente paso será conectar GitHub con un servicio gratuito llamado **Render** para que tu aplicación tenga un link real (como `https://tu-app.render.com`) que funcione 24/7.
